## Задание 3. Длительная задача с опросом статуса (polling)

**Цель:** Реализовать эндпоинт для запуска длительной задачи (имитация обработки изображения), возвращающий `task_id`. Клиент может опрашивать статус по `task_id` и получить результат после завершения.

1. Создайте `app/storage.py` – хранилище задач в памяти (словарь). Реализуйте функции:
   - `create_task()` – генерирует UUID, создаёт запись со статусом `pending`.
   - `update_task(task_id, status, result=None)` – обновляет статус и результат.
   - `get_task(task_id)` – возвращает задачу или `None`.   
![](./PrtSc/15.jpg)   
2. В `app/tasks.py` создайте асинхронную функцию `long_running_task(task_id, input_data)`. Имитируйте длительную обработку (например, 10 шагов с паузой 1 сек), обновляйте прогресс через `update_task`, по завершении – статус `completed` с результатом. Обработайте исключения, устанавливая статус `error`.   
![](./PrtSc/16.jpg)   
3. В `app/routers/tasks.py` добавьте новый роутер с префиксом `/tasks` и тегом `long tasks`. Определите эндпоинты:
   - `POST /process` – принимает JSON с входными данными, создаёт задачу, запускает `long_running_task` через `BackgroundTasks`, возвращает `task_id`.
   - `GET /{task_id}` – возвращает состояние задачи или 404.   
![](./PrtSc/17.jpg)   
4. Проверьте: создайте задачу (`POST /tasks/process` с JSON `{"name": "test"}`), опрашивайте статус. Должны видеть смену статусов `pending` → `running` (с прогрессом) → `completed`.   
![](./PrtSc/18_1.jpg)   
![](./PrtSc/18_2.jpg)   
![](./PrtSc/18_3.jpg)   
![](./PrtSc/18_4.jpg)   
5. Сделайте коммит:
   - `git add app/storage.py app/tasks.py app/routers/tasks.py`
   - `git commit -m "feat: добавлена длительная задача с опросом статуса"`

## Задание 4. Запуск синхронного кода в асинхронном контексте

**Цель:** Показать, как правильно вызывать синхронные блокирующие функции (CPU-bound вычисления) внутри асинхронного эндпоинта, не блокируя event loop, с помощью `asyncio.to_thread`.

1. В `app/tasks.py` добавьте синхронную функцию `sync_cpu_bound(data: dict) -> dict`. Используйте `time.sleep(3)` для имитации вычислений, возвращайте `{"computed": data.get("value", 0) * 2}`.   
![](./PrtSc/20.jpg)   
2. В `app/routers/tasks.py` добавьте эндпоинт `POST /tasks/compute`. Принимайте JSON, запускайте `sync_cpu_bound` через `await asyncio.to_thread(...)`, возвращайте результат.   
![](./PrtSc/21.jpg)   
3. Проверьте: `POST /tasks/compute` с `{"value": 21}` → ответ `{"computed": 42}`. Во время 3-секундной паузы сервер должен обрабатывать другие запросы.   
![](./PrtSc/22.jpg)   
![](./PrtSc/22_2.jpg)   
4. Сделайте коммит:

   - Примите соединение.
